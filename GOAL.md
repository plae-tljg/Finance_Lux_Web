# GOAL.md

目标：打造一个 Luxury 级别的财务管理 Web 应用，通过 OpenCode 自动生成代码并推送到 GitHub 仓库。

## 应用定位

一个专业级的浏览器端财务管理工具，提供精致的 UI、强大的功能、深入的报表分析和优雅的数据可视化。

## Luxury UI 标准

1. **视觉设计**
   - 精致的暗色/亮色主题，支持一键切换
   - 流畅的动画和过渡效果（页面切换、卡片悬浮、按钮点击）
   - 统一的图标系统，使用一致的 SVG/Icon 风格
   - 精致的卡片阴影和边框效果
   - 专业的配色方案，财务类应用的信任感

2. **布局设计**
   - 响应式布局，完美支持桌面/平板/手机
   - 优雅的侧边导航栏，可折叠
   - 数据看板（Dashboard）展示核心指标卡片
   - 清晰的信息层级和留白

3. **表格设计 (DataTable)**
   - 支持排序、筛选、搜索
   - 分页展示，可自定义每页条数
   - 行悬浮高亮，悬停显示操作按钮
   - 复选框批量操作
   - 列宽可拖动调整
   - 空状态优雅展示
   - 加载状态骨架屏

## Luxury 功能标准

1. **Dashboard**
   - 核心财务指标卡片（总收入/总支出/结余/预算使用率）
   - 本月收支趋势图
   - 分类支出占比（饼图/环形图）
   - 最近交易列表
   - 预算进度条

2. **交易管理 (Transactions)**
   - 交易列表：日期筛选、分类筛选、账户筛选、金额范围
   - 快速添加交易浮窗
   - 交易详情展开
   - 批量删除/编辑
   - 收支分类统计

3. **预算管理 (Budgets)**
   - 预算概览：总预算、已使用、剩余
   - 分类预算进度条
   - 预算使用率可视化
   - 超支预警（红色高亮）
   - 周期切换（日/周/月/年）

4. **报表分析 (Reports)**
   - 时间范围选择器
   - 收支趋势折线图
   - 分类支出对比柱状图
   - 收入来源饼图
   - Top 分类排行榜
   - 账户对比分析
   - 报表导出（PDF/Excel 格式）

5. **账户管理 (Accounts)**
   - 账户列表卡片式展示
   - 账户余额趋势图
   - 账户间转账
   - 账户余额编辑（月末记账）

6. **分类管理 (Categories)**
   - 分类树形/网格展示
   - 拖拽排序
   - 图标选择器

## 创意 Luxury 方向

当精确标准完成后，以下是更多 Luxury 方向可以探索：

1. **视觉元素**
   - 使用高质量背景图片（模糊、渐变效果）
   - 动态粒子/光晕效果
   - 玻璃态（glassmorphism）UI 风格
   - 精致的分隔线和装饰元素

2. **个性化元素**
   - 可爱的动画角色/宠物（如 Arknights 风格）
   - 节日主题装饰
   - 自定义光标效果
   - 开机欢迎动画

3. **动效设计**
   - 页面切换过渡动画
   - 卡片悬浮 3D 效果
   - 按钮点击反馈动画
   - 加载状态趣味动画

4. **彩蛋功能**
   - 隐藏的交互彩蛋
   - 成就系统/徽章
   - 签到奖励
   - 主题解锁

5. **社交化元素**
   - 分享功能（生成精美图片）
   - 表情包/贴纸
   - 备注心情/标签

## Luxury 数据表格标准

所有数据表格必须具备：
- ✅ 排序功能（点击表头排序）
- ✅ 筛选功能（列筛选器）
- ✅ 搜索功能（全局搜索）
- ✅ 分页功能（可配置每页条数）
- ✅ 列显示/隐藏切换
- ✅ 导出功能（CSV/Excel）
- ✅ 行操作（编辑/删除）
- ✅ 批量选择与操作
- ✅ 空状态展示
- ✅ 加载骨架屏

## 开发原则

1. **多语言尝试**：不要局限于一种语言。可以尝试 Python、JavaScript、Shell 等不同工具，选择最适合的项目。

2. **模块化设计**：代码要有良好的结构和模块划分，不要把所有功能堆在一个文件里。

3. **创意发挥**：可以自由添加有趣的功能和注释，但要有实际用途和可运行性。

4. **独立思考**：遇到困难时，尝试自行生成解决方案，而不是等待指令。

5. **Luxury 优先**：所有新功能必须先考虑 Luxury 标准，确保 UI 和功能都是高质量的。

## 工作流程

每次运行视为一个独立开发迭代：

1. 阅读 GOAL.md 了解当前任务
2. 检查 `NEXT.md` 是否有后续指示
3. 根据目标编写或修改代码
4. 确保代码可运行且有实际功能
5. 提交并推送到 GitHub

## 任务管理

每次完成一个功能或目标后：

1. 在 `NEXT.md` 中记录：已完成的工作、接下来的目标、新的创意想法
2. 如果当前任务完成，主动在 `NEXT.md` 中给自己设定新的目标
3. 如果 `NEXT.md` 为空或任务已完成，思考下一个合理的开发方向并更新它

## 注意

- 代码要有实际可运行的意义，不要生成"死代码"
- 功能要尽量完整，不要半途而废
- 保持提交记录清晰，描述完成的工作
- 每次开发优先考虑 Luxury 标准
